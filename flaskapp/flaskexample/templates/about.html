About Silicon Search

Silicon Search is an application that provides suggestions to tech companies for new cities to expand to.

Data

Data for about 250 metropolitan areas from the US Census's 2015 American Community Surveys was downloaded from IPUMS.org. The data, represented by county, were aggregated by metropolitan area.

Algorithm

Cities were labeled as "good for tech" vs. "less good for tech" using label propagation from Python's scikit-learn. Cities with over 3% of jobs in the tech industry were labeled as "Good for tech." The algorithm was validated through k-fold cross-validation. The features used for label propagation were percent of the population with a bachelor's degree and percent of the population between the ages of 18 and 35.

The similarity between cities was calculated using the Euclidean distance derived from data from the American Community Survey, including the features used for the label propagation algorithm as well as the diversity and price of rent.

The diversity index was calculated using the algorithm described here, from FiveThirtyEight. Diversity scores range from 0.0 to 0.8, with 0.8 being the most diverse.

Tools

For the back-end, I used Python (pandas) and R (dplyr, tidyr, ggplot2) to explore the data, and Python's scikit-learn for label propagation and calculating a distance matrix between the metropolitan areas. Aggregated data is stored in MySQL tables, using the pymysql package in Python for the interface.

The web app is designed using HTML and Python's Flask and deployed on Amazon Web Services. Data is queried using a live MySQL call within the Flask app. D3 is used to visualize the data on a United States map.
